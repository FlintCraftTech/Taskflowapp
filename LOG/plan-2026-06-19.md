This session processed all five queued captures, most of them fallout from the aborted device-verify test session. The spine of the work was the project-creation gap: the app assumes Projects exist but has no way to create one, which blocked the Project device-verification and is a real missing feature.

Mid-discussion the user floated a larger redesign — making Projects abstract (no Project screen), reached only via the Strategy doc or a project-name search, possibly dropping Later. That alternative was weighed and rejected: removing the Project screen would delete the one place undated, uncommitted tasks can be browsed (the divergent "dreaming surface", UX principle 1), leaving them findable only by searching — the out-of-sight-out-of-mind failure the app exists to fight for its executive-dysfunction users; and the "focus on one project" goal driving the idea is already covered by the parked Search feature, so the redesign would trade away a core surface to solve an already-solved problem. Projects stay as screens; the redesign is recorded here so it isn't re-proposed.

With that settled, project-creation became a name-only gesture — a spec-edit ([project-create-spec-edit], adding the creation behaviour to §Projects overview and tightening §Add's "from any screen" wording, since both touch the Projects-overview surface) plus a feature batch ([project-create]: the "+" affordance, insert, and empty Strategy entry, with its own Test section). The description is deliberately not captured at creation — it is the Strategy-doc paragraph, authored later (UX principle 5).
