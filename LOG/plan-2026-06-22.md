# [HASH] — /plan: queued the Later-by-Project redesign as three top-of-Build batches (spec-edit, system-Unassigned data model, Project-grouped Later screen removing the per-Project view + overview), resolving both 2026-06-21 device-test captures; rescoped [task-reorder-within-list], dropped two obsolete deferred tests, parked the execute-by-area question

Alex chose to pull [later-by-project] — the Later + Projects-overview merge — off the post-core shelf and build it now, turning this /plan into a design-resolution pass that settled its open questions and queued the work. The redesign reworks the right end of the spine into one "zoom by time" flow: Later stops showing far-future tasks by date and becomes a list of expand/collapse Project cards (ordered to match the Strategy doc, the system "Unassigned" card pinned bottom), and the standalone Projects overview page and per-Project view both go away. This resolves the two captures from the 2026-06-21 device test — Projects leave the side menu, and there's no separate Project surface to "whisk you off the spine," because a Project is now just a card on Later.

Load-bearing decisions, with the alternatives that lost:
- Every task always has a Project. An explicit system "Unassigned" Project (no null projectId) replaces the null/no-Project state. Alex proposed this to kill orphan tasks that render nowhere; chosen over a null + virtual-section approach because Later-by-Project must render every task under a Project section, so one uniform group-by-Project path is cleaner. The Unassigned Project is system-flagged: undeletable, excluded from the Strategy doc, pinned bottom.
- Moving a task between Projects stays, via the editor's Project picker, identical in free and paid. Alex first proposed no refile at all (delete-and-rewrite only, on the premise that Claude authors coherent projects), then reversed it: free users have no Claude help and must move by hand, so paid mustn't lose it either, and delete-and-rewrite destroys a task's subtasks, notes, and recurrence. So the per-Project view is removed but 0005's Project picker is kept.
- No Project-specific "add" affordance. An in-card "+" on Later was rejected — Alex's reason: it multiplies the ways one thing is done and the ways "+" is mis-pressed. Tasks are added the single existing way (slot FABs); a task joins a Project via the editor.
- Soon stays date-grouped; only Later becomes Project-grouped — the zoom metaphor, area over date at the far horizon.

The execute-by-task-area question Alex raised — a Project's tasks visible under Today/Tomorrow/Soon, not just Later — has no clean answer yet and was parked against the screen build, explicitly not a blocker. Terminology note: "someday" is not a Taskflow bucket; undated tasks are a property, and the ones that lived below the Project-view card now move into the Project's Later card.
